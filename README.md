## The prompt:

```
*Situation*
In today's complex financial landscape, individuals and businesses need an efficient way to track and manage their expenses systematically and comprehensively.

*Task*
Design a fully functional expense tracker application with the following core features:
- Create a user-friendly interface for expense logging
- Implement multiple expense categorization options
- Develop robust financial reporting and visualization capabilities
- Enable budget setting and tracking
- Support multiple currency inputs
- Provide data export and backup functionality

*Objective*
Develop a comprehensive expense tracking solution that helps users:
- Gain clear insights into their spending patterns
- Make informed financial decisions
- Manage personal or business budgets effectively
- Reduce unnecessary expenditures
- Simplify financial record-keeping

*Knowledge*
Key application requirements:
- Support manual and automatic expense entry
- Categorize expenses by type (e.g., food, transportation, utilities)
- Generate monthly/yearly financial reports
- Implement secure user authentication
- Create visual representations of spending (charts, graphs)
- Allow receipt image uploads and OCR scanning
- Provide predictive spending analysis
- Ensure data privacy and encryption

*Examples*
Example 1:
"""Expense Entry Format:
Date: 2023-11-15
Amount: $45.67
Category: Dining
Payment Method: Credit Card
Notes: Business lunch meeting"""

Example 2:
"""Monthly Report Structure:
Total Expenses: $2,345
Top Categories:
1. Housing: $850
2. Transportation: $350
3. Food: $475
Budget Compliance: 92% (On Track)"""

The assistant should:
- Create a modular, scalable application architecture
- Prioritize user experience and intuitive design
- Implement robust error handling and data validation
- Ensure cross-platform compatibility (web/mobile)
- Provide real-time expense tracking and notifications
- Support multiple user roles (individual/family/business)
- Use secure, encrypted data storage methods
```
